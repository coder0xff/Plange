A property is a symbol representing a relation or get and set accessors defined in another execution context. A purely functional Property has no get or set accessors, but is defined using an expression. There are also imperative properties with explicitly implemented accessors. Semantically, the defining Scope becomes accessible by the caller (the code accessing the value), but only through code provided by that Scope.

a pure property 

                Angle := class inheriting {
    <Double> radians;
    <Double> degrees := radians / Math.pi * 180;
}
Leveraging a computer algebra system, get and set accessors are simulated/generated if needed (and if able).

an imperative property 

                <Double> v;
clamp := {
    get { return v; }
    set { v <- value < 0 ? 0 : value > 1 ? 1 : value; }
};
Note that both of these forms can be used within a class, or as a top-level statement. Read only and write only properties can be achieved by implementing only the get or set accessor, respectively.Â  Additionally, the imperative syntax provides one to three additional Scopes. The second example, now with v encapsulated: 

                    clamp := {   
    <Double> v;
    get { return v; }
    set { v <- value < 0 ? 0 : value > 1 ? 1 : value; }
};
The most common use of properties will likely be in assignments: 

                <Angle> angle;
                

                

                angle.degrees <- 45; //assignment operator
However, properties have members as well. In the case of a pure property, the object inherits from Relation. In the case of an imperative property, it is an object with two members, get and set.
